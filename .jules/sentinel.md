## 2024-05-18 - [Add Content Security Policy (CSP)]
**Vulnerability:** Missing security headers (CSP) allowing any resources to be loaded, putting the application at risk for Cross-Site Scripting (XSS) and data injection attacks.
**Learning:** This application heavily relies on several specific external resources, such as Google Analytics, OpenStreetMap tiles, and Google Fonts. A strict CSP must explicitly allow these origins (`https://www.google-analytics.com`, `https://*.tile.openstreetmap.org`, `https://fonts.googleapis.com`, `https://fonts.gstatic.com`). Furthermore, because it's a React app using inline styles (or certain styling approaches like Tailwind) and potentially inline scripts during development/build tools, 'unsafe-inline' is often practically required in the `style-src` and `script-src` directives unless strict nonces/hashes are configured.
**Prevention:** Implement a strong Content Security Policy early in development and update it systematically as new external dependencies are added, ensuring to restrict `default-src` to 'self' and only open up specific directives as absolutely necessary.
